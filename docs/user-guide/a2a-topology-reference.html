Agent-to-agent communication across process boundaries
01Local vs Remote Comparison
Local Agent IN-PROCESS
  • In-process, shared memory
  • Direct state access via session.state
  • Millisecond latency
  • Exception on failure
  • No network needed
Agent("helper", "gemini-2.5-flash")
Remote A2A Agent CROSS-PROCESS
  • Separate process / machine
  • State bridged via .sends() / .receives()
  • Seconds latency (network + remote LLM)
  • HTTP timeout, 503, FAILED task
  • Needs retry, circuit breaker, timeout
RemoteAgent("helper", agent_card="http://...")

04Resilience Patterns
Retry
M.a2a_retry(max_attempts=3)
Exponential backoff between retries. Configurable max attempts and delay multiplier.
Circuit Breaker
M.a2a_circuit_breaker(max_fails=5)
Trips open after consecutive failures. Auto-probes after cooldown. Prevents cascading failure.
Timeout
M.a2a_timeout(seconds=30)
Hard time limit per A2A call. Prevents hanging on unresponsive remote agents.
05Composition Examples
Sequential pipeline with remote agent
# Remote reviewer in a local pipeline pipeline = Agent("writer") >> remote_reviewer >> Agent("editor")
Fallback chain: fast remote → slow remote → local
# Cascading fallback across boundaries chain = RemoteAgent("fast") // RemoteAgent("backup") // Agent("local")
Parallel fan-out mixing local and remote
# Local and remote agents in parallel fanout = Agent("local-search") | RemoteAgent("web-search") | RemoteAgent("paper-search")
Remote agent as tool
# Expose remote agent as a callable tool agent = Agent("coordinator").tools(T.a2a("http://research:8001/agent.json"))
06Discovery Methods
DNS Well-Known
Discover agents via DNS-based well-known URI resolution. Zero-config for agents published with standard conventions.
RemoteAgent.discover( "research.agents.acme.com" )
Registry
Query a central agent registry by skill, capability, or metadata. Supports dynamic topology updates.
AgentRegistry( "http://registry:9000" ).find(skill="research")
Environment
Read agent card URL from environment variable. Ideal for container deployments and CI/CD pipelines.
RemoteAgent( "code", env="CODE_AGENT_URL" )
